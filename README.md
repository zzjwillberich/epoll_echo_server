# epoll + 线程池 Echo 服务器

> 大一学生在 Linux 上的 C++ 网络编程实践项目。

## 项目简介

基于 **epoll (ET 模式)** + **自建线程池** 的高并发 Echo 服务器。支持：

- epoll ET 边缘触发模式，高效事件通知
- 自建线程池，异步处理客户端请求
- 非阻塞 I/O + 循环读写，正确处理 EAGAIN
- 写缓冲区机制——写不完的数据暂存，等待 EPOLLOUT 事件续写
- 优雅的客户端断开处理
- CMake 构建系统

## 关键技术点

### epoll ET 模式
- 使用边缘触发，只在状态变化时通知一次
- 必须配合非阻塞 I/O，循环读写直到 EAGAIN

### 自建线程池
- 基于 std::thread + std::queue + std::mutex + std::condition_variable
- 任务队列 + 工作线程模型
- 优雅关闭：析构时通知所有线程，等待任务完成

### 异步写缓冲区
- write() 可能无法一次写完所有数据（EAGAIN）
- 未写完的数据暂存到 unordered_map 缓冲区
- 注册 EPOLLOUT 事件，下次可写时继续发送
